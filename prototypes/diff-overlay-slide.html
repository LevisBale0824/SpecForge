根因：v-if 销毁 DOM + flex row 宽度变化 → 数千节点 reflow。

User

overlay 会遮挡，有什么更好的方案？

Agent

把 diff 提升到 App 层作为右侧独立 drawer，和 sidebar 同级：

App 层布局: <div class="app-row"> <SidePanel /> ← 左 sidebar <main class="center flex-1"> ← ChatView + InputPanel <ChatView /> <InputPanel /> </main> <div class="diff-drawer"> ← ★ 右侧独立 drawer <SessionDiffPanel /> </div> </div> 关键: ChatView 代码一行都不用改。createApp → provide showDiffPanel → diff-drawer 消费。ChatView 内部 layout 完全不参与 diff 面板的宽度计算。
User

就是说 diff 不在 ChatView 里面了？

Agent

对。diff 面板从 ChatView 的内部 flex 子元素 提升为 App 层的独立列。ChatView 只管渲染消息，宽度由 App 层给出。开关 diff 只影响 App 层的 flex 分配，不触发 ChatView 内部重排。

按 Ctrl+Shift+D 试试——diff 从右侧滑入/滑出，消息气泡正常显示，输入框始终可见。
